# PiezoJet 技术审计说明

**用途：** 本文是给外部研究者或代码审计模型的技术交接材料。它记录截至 2026-07-15 的数据谱系、张量约定、实际代码路径、训练/评估协议、已完成实验和已知问题，便于逐条验证或提出替换方案。

本文严格区分三类表述：

- **实现事实**：可以在本仓库源代码或已保存产物中直接复现的行为；
- **实验事实**：固定数据、固定脚本下已经得到的数值结果；
- **待审计假设**：代码试图施加的物理结构，但尚不能仅凭现有实验认定为物理正确。

这不是性能宣传材料。目前最重要的实证结论是：完整内部应变标签和较多样的训练数据能改善部分 ionic-path 诊断，但**总压电响应相对零预测器仍为负 skill**；模型不能被称为已经学会一般的总压电响应。

## 1. 项目目标与边界

PiezoJet 的目标不是只从晶体结构直接回归一个 `3 x 6` 压电张量，而是尝试显式预测原子坐标层面的线性响应因子，并通过光学子空间的响应算子组合为晶格介导（ionic）压电项：

\[
e^{\mathrm{total}} = e^{\mathrm{electronic}} + e^{\mathrm{ionic}},\qquad
e^{\mathrm{ionic}} = \frac{16.02176634}{V}Z^{*\mathsf T}\,O(\Phi)\,\Lambda.
\]

这里 `Z*` 为 Born effective charge，`Phi` 为力常数矩阵，`Lambda` 为内部应变/应变力耦合，`V` 为晶胞体积，`O(Phi)` 是在去平移的光学子空间上的算子。模型也同时产生介电和弹性响应，但当前的核心审计对象是上述压电分解。

项目坚持以下边界：

- 主监督和当前基准均为 **JARVIS/GMTNet 同源数据**；没有把 Materials Project 标签混入 JARVIS-only 测试；
- 不用不完整 OUTCAR 区块猜测完整 `Lambda`；完整标签必须通过独立的严格补全审计；
- 不会为了增加样本数降低严格门槛；
- 冻结的 20 个测试材料不参与样本选择、超参数选择或检查点选择；
- No.187 (`P-6m2`) 只从严格补全中隔离，不能据此断言上游 JARVIS 记录错误；其有效的已打印部分标签仍可作 masked supervision。

## 3. 数据来源、身份与版本

### 3.1 宏观压电主标签

原始索引位于 `data/raw/gmtnet/data/jarvis_diele_piezo.pkl`，由 GMTNet 发布的 JARVIS-DFT 压电数据提供。加载函数 `load_gmtnet_records` 只接受以下记录：

- 有 `JARVIS_ID`、`atoms`、`piezoelectric_C_m2`；
- 压电数组形状为 `3 x 6`、所有数值有限；
- `abs(value).max() < 100`，用于剔除明显无效值。

当前索引包含并通过上述 schema 筛选的 **4,998 / 4,998** 条记录。每条结构字段包括元素序列 `atoms["elements"]`、分数坐标 `atoms["coords"]` 与晶格矩阵 `atoms["lattice_mat"]`。主标签单位为 `C/m^2`。

全局 GMTNet 训练/验证/测试划分是按化学式分组、并按响应范数分层的约 `4000 / 499 / 499`。它只适用于宏观压电任务，**不是**后文严格完整因子的 69/10/20 基准。

### 3.2 同源 JARVIS 原始 DFPT 档案

为了获得原子层面的因子，代码以同一 `JVASP-*` ID 从 JARVIS 公共 Figshare `raw_files` 的 `DFPT` 档案检索 VASP 原始文件。该路径由 `jarvis.db.figshare`、`jarvis.io.vasp.outputs.Vasprun` 和 `Outcar` 驱动；不依赖私有 Materials Project API，也没有使用任何私密 key。

每个档案要求同时具备一个 `vasprun.xml` 和一个 `OUTCAR`。解析并缓存的字段包括：

- `born_charges`：`N x 3 x 3` Born effective charge；
- `force_constants`：`N x N x 3 x 3` 的 Gamma 点力常数；
- `dynamical_eigenvalues` / `dynamical_eigenvectors`：解析器给出的 Gamma 动力学矩阵本征对；
- `masses`、原始 dynamical matrix、介电字段；
- OUTCAR 打印的 ionic/total piezo `3 x 6`；
- OUTCAR 中实际打印的内部应变区块、离子编号、笛卡尔方向。

下载后先验证档案末态与 GMTNet 记录的元素顺序、晶格和周期等价分数坐标均一致，容差 `2e-5 Å`。有元素顺序、晶格、坐标或文件格式不一致时，记录会失败而不是被重排或静默修复。例如第二轮中 `JVASP-1048` 因 archive 坐标不一致拒绝，`JVASP-28581` 因 malformed internal-strain OUTCAR block 拒绝，二者均没有作为标签使用。

DFPT cache schema 为 2；当前最终同源并集为 `data/processed/jarvis_dfpt_v3/`，共 **610** 个缓存材料。缓存保存的是已验证的解析 tensor 和 source archive 名，而不是另一个来源的代理标签。

### 3.3 完整内部应变的严格补全

VASP OUTCAR 常只打印对称不等价的内部应变区块，不能直接视作完整 `N x 3 x 3 x 3` 标签。`strain_completion.py` 不做启发式填充，而是把全量张量限制在晶体空间群不变子空间中，只有下列所有条件同时通过才输出完整标签：

1. 用 `pymatgen.SpacegroupAnalyzer` 生成笛卡尔空间群操作；按同种元素 Hungarian assignment 建立双射原子置换；
2. 施加三阶 `Lambda[kappa, alpha, j, k]` 的空间群不变性，且 `j,k` 对称；
3. 施加 acoustic sum rule：每个 displacement direction 与应变分量上，所有原子的内部力之和为零；
4. 最大原子映射误差不超过 `2e-5 Å`；
5. 已打印区块在不变基上的观测矩阵满秩，因而唯一确定所有不变自由度；
6. 已观测区块的相对拟合残差不超过 `5e-3`；
7. 若有冗余区块，leave-one-block-out 最大相对残差不超过 `5e-2`；
8. 使用档案的真 `Z*`、真 `Phi` 和完成的 `Lambda`，按同一 regularized 光学算子重建 OUTCAR ionic piezo；相对 closure error 不超过 `5e-2`。

因此 strict-complete 是一个很强的质量标签，而非“所有可下载档案”的同义词。失败样本仍保留其已打印的 masked 区块，但不获取 full-Lambda loss。

版本沿革：

| 数据版本 | strict complete 数 | 说明 |
| --- | ---: | --- |
| `jarvis_strain_completion_v4` | 99 | 初始、schema 验证后的不可变基础集 |
| `jarvis_strain_completion_v5` | 127 | 第一轮 information-gain 队列新增 28；该队列严重 cubic-heavy |
| `jarvis_strain_completion_v6` | 138 | 当前并集；第二轮 coverage-aware 队列新增 11 |

第二轮检索只在“下载候选”阶段按冻结测试集晶系比例配额：cubic 20、hexagonal 20、orthorhombic 20、tetragonal 15、trigonal 10、monoclinic 10、triclinic 5；不改变任何 strict gate。100 个候选中 98 可审计，11 个通过（5 cubic、3 hexagonal、2 tetragonal、1 orthorhombic）；trigonal、monoclinic、triclinic 均没有通过。非互斥失败计数为 `invariant_fit=57`、`redundant_block=41`、`ionic_closure=52`、`not_uniquely_identifiable=9`。这说明低对称覆盖仍是数据瓶颈，不能因为检索配额存在就声称已有低对称 strict coverage。

## 4. 张量、单位与约定

这是最需要逐行审计的部分。所有压电转换集中在 `tensor_ops.py`，避免不同模块各自重写 Voigt 顺序。

- 源 GMTNet/JARVIS `3 x 6` 列顺序：`(xx, yy, zz, xy, yz, xz)`；
- 内部 canonical 顺序：`(xx, yy, zz, yz, xz, xy)`；
- 使用 engineering strain：`(eta_xx, eta_yy, eta_zz, 2 eta_yz, 2 eta_xz, 2 eta_xy)`；
- Cartesian 压电存为 `e[i,j,k]=e[i,k,j]`，数组为 27 分量、独立物理分量为 18；
- `Lambda[kappa, alpha, j, k]` 同样在最后两维对称；其实现中的 internal tensor 是从 canonical engineering-Voigt coupling 填到对称 `3 x 3` 尾维；
- `Phi[kappa,alpha,kappa',beta]` 单位 `eV/Å^2`；`Lambda` 单位 `eV/Å` per unit strain；`Z*` 的行是原子 force/displacement direction，列是 electric-field/polarization direction。

关键换算常数在 `AtomCoordinateResponsePotential`：

- piezo：`16.02176634`，目标单位 `C/m^2`；
- dielectric：`180.9512817`，目标为相对介电常数；
- elastic：`160.2176634`，`eV/Å^3 -> GPa`。

代码将预测 BEC 按每个图减去原子均值，强制 `sum_kappa Z*[kappa] = 0`。DFPT 真 BEC 也在数据加载时作同样零和投影，并记录原始 ASR 残差/投影相对量，故训练目标不是未修正的原始 array。这是明确的物理先验；是否应以质量加权、是否应保留更丰富的 charge-neutrality 误差模型，值得审查。

**重点待核查：** VASP 的 force-constant、BEC 与 internal-strain 的 index/sign convention 是否恰与上述矩阵乘法一致；engineering shear 是否在 `Lambda` 的读入、能量二阶导数、OUTCAR ionic-piezo closure 三处完全一致；常数以及 `V=abs(det(cell))` 的晶格行/列约定是否正确。不得仅凭某些 closure 通过就将全局 convention 判定为正确。

## 5. 数据如何变成模型输入

### 5.1 周期图

`data.py` 以分数坐标、晶格矩阵、cutoff 构建有限邻居 PBC 图。边相对向量始终按

\[
r_{source} - r_{target} + \mathrm{edge\_shift}
\]

计算，其中 `edge_shift` 已转成笛卡尔坐标。默认 `cutoff=5.0 Å`，每中心原子最多 `32` 个邻居。图保存 `pos`、`cell`、`edge_index`、`edge_shift`、原子序数、批次索引及所有有 mask 的标签。

这套图构建没有直接采用 ASE/PyG 的现成 periodic neighbor list。实现相对短，但 PBC image 选择、同一原子多 image、cutoff 边界、晶格 row/column convention 是高优先级审计项目。

### 5.2 训练样本携带的监督

每个样本均可有宏观总压电 target `y`；若相应 DFPT cache 存在，还携带 BEC、force constants、Gamma modes、OUTCAR ionic piezo 和真实打印的内部应变区块。若相应 strict completion 存在，才额外携带 `dfpt_internal_strain_full` 与 full mask。缺失项均为 mask，不用零值伪装成标签。

训练前还进行结构自监督：masked-species 与去除整体平移的坐标加噪/denoising；默认 `mask_probability=0.15`、`coordinate_noise_std=0.02`。主实验加载该 Cartesian encoder 的预训练 checkpoint；外部审计应核查预训练数据是否严格不包含冻结测试材料，及其 checkpoint 的生成谱系。

## 6. 模型：真实代码结构与输入特征

`PiezoJet` 的当前 production encoder 是 `CartesianLocalEnvironmentEncoder`，**不是**文件中仍保留的历史 `PeriodicCrystalEncoder`（e3nn spherical harmonics 版本）。模型配置为：embedding 32、3 blocks、12 个 Gaussian radial basis、radial hidden 64、48 vector/quadrupole channels、global context 128、reciprocal cutoff 7.0、默认 `factor_architecture=energy_learned_strain`。`star_rank=4` 在该默认架构下不启用；只有 `energy_learned_star` 才会用它。

### 6.1 局部 encoder 特征

每个节点/边使用以下信息：

1. 原子序数 `Z` 的 `Embedding(119, 32)`；
2. PBC 边距离和 12 维 Gaussian radial basis 加 cutoff envelope；
3. 三层 invariant chemical message passing，更新每原子的 scalar feature；
4. edge unit vector `n` 和 traceless rank-2 tensor `nn^T-I/3`；
5. scalar state 产生 edge coefficient，把 `n` 与 quadrupole 聚合为每节点 48 通道的 vector 和 quadrupole features；
6. 一个可学习 `48 x 48` 的 channel-space interaction matrix 做通道混合；degree exponent 是可学习标量并 clamp 到 `[0,1]`。

encoder 输出 `CartesianNodeFeatures = (scalar, vector, quadrupole)`。它试图保持 Cartesian `O(3)` 等变结构，但没有 e3nn 风格的完整 Clebsch--Gordan tensor-product 通道传播。该选择的表达力、反射/奇偶性、higher-order coupling 是否充分，不能仅凭命名为“equivariant”便认为已经证明。

### 6.2 局部 heads

- `CartesianPiezoTensorHead`：由 vector、quadrupole 和 identity 组合 rank-3 bases，强制最后两个应变 index 对称；节点 tensor mean pooling 为晶体 direct/electronic branch。
- `CartesianBornChargeHead`：组成 `isotropic I + symmetric-traceless quadrupole + antisymmetric Levi-Civita/vector`，随后每图去均值，精确施加 BEC ASR。
- `CartesianPolarReadout`：用 scalar gate 和 vector channels 输出每原子的 local polar vector。

### 6.3 全局上下文与谱项

`CrystalGlobalContext` 将下列不变量输入 MLP：元素 composition histogram、`log(volume)`、`log(density)`、physical reciprocal-shell 下的 learned chemical spectrum、local polar power spectrum、polar--chemical cross spectrum 以及 PBC 局部 polar fluctuation correlation。reciprocal vector shell 按物理缩放长度 `|2π h L^{-T}| V^(1/3)` 枚举，以试图使整整数晶格基变换只导致重索引。

同一模块还从 origin-invariant 的 polar--chemical cross spectrum 生成 rank-3 `spectral_operator`。最终电子项是：

\[
e^{\mathrm{electronic}} = \mathrm{sym}_{j,k}(e^{\mathrm{direct}} + e^{\mathrm{spectral}}).
\]

审计者应特别测试 reciprocal shell 在非正交 cell、交换/剪切晶格基、原点平移、不同等价 conventional/primitive cell 下是否真的不变，以及 cross-spectrum 的张量变换和 parity 是否正确。

### 6.4 `Phi` 与 `Lambda` 的共享能量因子

默认 `StrainAwareQuadraticEnergyHead` 为每条 directed periodic bond 输出一个 signed symmetric `3 x 3` stiffness `K_e`，由 scalar pair features、radial basis、全局 context 和平均 local polar 构造。其代码声明共享的局部二次能量：

\[
U_e = \frac12(B_eu + S_e\eta)^{\mathsf T}K_e(B_eu+S_e\eta),
\]

从而使用同一 `K_e` 组装：

\[
\Phi=\sum_e B_e^{\mathsf T}K_eB_e,\qquad
\Lambda=-\sum_eB_e^{\mathsf T}K_eS_e,\qquad
K_{\eta}=\sum_e S_e^{\mathsf T}K_eS_e.
\]

基础 `S_e` 是对每个 unit engineering strain 的 affine bond strain；`energy_learned_strain` 再加由 `n`、local polar、identity 构成的七类 rank-3 basis correction，且最后层零初始化。组装 `Phi` 时加入 bond 的 `(+,+,-,-)` Laplacian block，显式对称化，故 uniform translation 被消去；`Lambda` 为 source/target 的反号积累。`energy_learned_star` 另外有可选低秩 star term，但默认架构未使用。

这是非常重要的**代码形式**：在固定网络输出下，主 bond 项确实按共同二次型组装。可是“这个可学习、可能为负的 bond stiffness + learned strain correction 是否仍等同于真实晶体势能的二阶导数”“directed edge 是否双计数”“global context 依赖是否破坏期望的局域/可加性”等都尚未完成物理证明。应要求审计者从自动微分或有限差分构造可控 toy case 验证 integrability、Hessian symmetry、ASR 与各项符号。

### 6.5 光学响应、介电与弹性分支

`AtomCoordinateResponsePotential` 先构造正交 projector，去掉恰好三条 uniform Cartesian translation。对 projected force matrix，生产路径使用 signed regularized Green filter：

\[
O_\delta(\Phi)=Q\,\Phi(\Phi^2+\delta^2Q)^{-1}Q,qquad
f_\delta(\lambda)=\frac{\lambda}{\lambda^2+\delta^2},\quad \delta=10^{-3}\ \mathrm{eV/Å^2}.
\]

它保留负/不稳定 mode 的符号、在软模附近有界、并使梯度连续；但它**不是**无正则二次势能的 stationary inverse。exact inverse 只作为稳定、非奇异情形的诊断。生产训练、strict closure 和 canonical ionic evaluator 使用同一个 regularized policy，这是内部一致性，不等于这个 regularizer 已被证明是最合适的物理 observable。

响应分支为：

\[
\begin{aligned}
e^{\mathrm{ionic}}&=c_eV^{-1}Z^{*\mathsf T}O\Lambda,\\
\epsilon&=\epsilon_{\mathrm{background}}+c_\epsilon V^{-1}Z^{*\mathsf T}OZ^*,\\
C&=C_{\mathrm{background}}+c_CV^{-1}(K_\eta-\Lambda^{\mathsf T}O\Lambda).
\end{aligned}
\]

`LinearResponseBackground` 以 global context 预测 isotropic non-negative Lamé/shear 及 susceptibility，因此 background elastic/dielectric 是强限制的各向同性形式。`PiezoJet.predict_components` 将 direct piezo、谱项、ionic term 相加。外部审计必须判断：

- electronic/direct 分支与 ionic/shared-strain 分支是否双计数或漏项；
- predicted `K_eta`、background elastic 和 relaxed elastic 的能量分解是否正确；
- 有符号 regularizer 用于 dielectric/elastic 时的正定性、稳定性及物理解释；
- 5 Å 局部图和有限 reciprocal representation 是否足以表示长程 Coulomb/phonon 耦合。

## 7. 训练目标和选择协议

默认联合训练有总压电损失，以及可用时的 dielectric、BEC、force constants、已打印 internal-strain、strict full-Lambda、response-active Lambda、soft mode、ionic piezo 损失。`config.yaml` 中大部分附加任务默认权重为 `0.1`；mode-aware Lambda loss 默认 `0.0`，即关闭。

response-active loss 用**真 DFPT `Z*` + 真 DFPT `Phi` + 预测 `Lambda`**构成 observable ionic product，隔离 Lambda 对 ionic response 的贡献。full-Lambda loss 只施加于 strict-complete 材料。mode-aware loss 针对近简并光学子空间实现了 gauge-safe 的 response-weighted 版本，但其一 seed smoke 没有改善 predicted ionic skill，故未作为 production default。

direct-factor curriculum 的因子期理论权重为 BEC `1`、force `1`、printed internal `5`、full Lambda `1`、soft `1`、response-active `1`、mode-aware `0`。较大的 `5` 是梯度尺度补偿，不是对标签重标度。

两个不同协议必须区分：

- 主 production learning curve：100 个固定 gradient updates，`batch_size=128`（这些小 cohort 每 epoch 约一 update），无 early stop，以 validation total loss 选 checkpoint；
- direct-factor B protocol：50 个 direct-factor updates，恢复 validation-factor-loss 最优点，随后 50 个 **冻结 factor stack** 的 joint updates；joint checkpoint 仍只按 validation loss 选。每个研究点使用 seeds 42/43/44。

训练或超参数选择从不读取 20-material test metrics。`reevaluate_protocol_checkpoints.py` 只读取已选的 `loss_best.pt` 进行 post-selection replay，不训练、不改 checkpoint。

## 8. 评估器与指标含义

DFPT evaluator 特别避免只报所有 tensor component 拼接后的“微平均”cosine。每份 canonical 报告至少区分：

- **full-Lambda macro cosine / amplitude**：逐材料比较 full strict `Lambda` 后再平均；
- **oracle ionic**：真 `Z*`、真 `Phi`、预测 `Lambda`，同一 regularized operator；隔离 Lambda 学习；
- **predicted ionic**：预测的所有因子组成 ionic branch；
- **macro cosine**：先按材料算 cosine 后平均，是主泛化指标；
- **micro cosine**：把所有 component 拼接后算，仅用于发现大样本/大范数材料的聚合偏置，不能替代 macro；
- **active-only ionic cosine**：只保留真实独立 18 分量 ionic Voigt norm 大于 `0.05 * sqrt(18) C/m^2` 的材料；
- **skill vs zero / total TRS**：相对全零预测的 MAE 型 response tensor skill，正值才优于零；
- branch norm、total norm 与 cancellation ratio：诊断量，不是正确性分数。

为防止接近零 target 使 ratio 失真，`evaluate_dfpt.py` 中 `FACTOR_FLOORS` 对 ionic/total/electronic 等采用 `0.05 C/m^2` 的分母下限。外部审核需复核 floor 和 active threshold 是否预注册且适合数据分布；不同 floor 对结论的敏感性也应报告。

### 9.2 优化干扰 forensics（A--G，固定 69 labels）

这些实验用于诊断优化，不是生产性能表。A 为 direct-factor only；B 是 50 factor + 50 frozen joint；C 同样恢复但 joint 阶段允许 factor 被改写；D 为交替训练；E 为 100 factor + 50 frozen joint；F/G 是单侧 factor-protected gradient projection，G 额外对 response gradient unit-norm matching。

- A：full-Lambda cosine `0.326 ± 0.067`；
- B：`0.210 ± 0.058`；
- C：`0.067 ± 0.162`。在 seed 42，factor stage validation cosine `0.387` 在 unfrozen joint 后降至 `0.178`，冻结版本为 `0.362`；这是 factor-path interference 的直接证据；
- E：尽管保持 factor-only full-Lambda 表现，total TRS `-2.179 ± 0.395`，是明确负结果；
- F：full-Lambda `0.142 ± 0.027`、total TRS `-0.038 ± 0.030`；
- G：full-Lambda `0.228 ± 0.099`、total skill `-0.141 ± 0.137`，ionic skill 仍非正。

五个 response-matched 35-label resamples 的 cosine 范围 `0.048--0.258`，说明小规模材料组成也是显著不确定来源。mode-aware loss 的一 seed smoke 也未提高 predicted ionic skill。上述结果不支持“靠更长 factor pretraining 或 gradient surgery 就已解决响应问题”。

## 10. 目前可以与不可以作出的结论

**可以作出的结论：**

1. 数据管线具有明确来源和 ID/结构核验；不完整 Lambda 没有被静默当成完整标签。
2. strict completion 的质量门控很苛刻，扩充不能靠放宽阈值获得；低对称样本正是最难通过的区域。
3. 把因子学习和 joint macro response 分开可暴露强烈的梯度干扰；冻结 factor stack 在当前小样本设置下比允许其被 joint loss 改写更稳定。
4. micro component cosine 可以与 material-macro conclusion 相矛盾，后者应作为主报告。
5. coverage-aware 的 108 训练集提高 ionic 相关诊断，但总响应尚未胜过 zero predictor。

**当前不能作出的结论：**

1. 不能宣称 PiezoJet 已实现可靠、可泛化的压电预测，尤其不能对低对称材料作此声称。
2. 不能将 signed regularized Green filter 解释成无条件真实的 phonon inverse 或静态平衡响应。
3. 不能把 form-in-code 的共同二次能量宣称为对真实 DFPT 能量面的充分物理建模。
4. 不能把 direct/factor 或任何 A--G 负对照调成新默认，只因某个局部指标变好。
5. 不能因 No.187 的冲突推断 JARVIS 数据“错误”，也不能施加未经控制重现的全局 Voigt/shear/sign patch。

## 11. 请求外部 GPT/审计者逐项回答的问题

请审计者给每项结论标为“正确”“可疑，需要单元测试”“实现错误”“可由成熟库安全替代”，并尽量给出最小可运行反例或测试方案。

### A. 数据与 DFPT 约定

1. `jarvis_dfpt.py` 从 `Vasprun.dfpt_data`、`phonon_data(fc_mass=False)`、`Outcar.piezoelectric_tensor` 提取的数组，在当前 jarvis-tools/VASP 版本下分别是什么 index、单位、mass weighting 和符号？`force_constants` 与 `dynamical_matrix` 是否被正确区分？
2. `OUTCAR` regex 对 `INTERNAL STRAIN TENSOR FOR ION ... DIRECTION ...` 的行列方向、单位、engineering shear 处理是否正确？应否改为官方 parser 或更健壮的 parser？
3. `source_voigt_to_canonical`、`piezo_voigt_to_cartesian`、`StrainAwareQuadraticEnergyHead._internal_tensor` 三处是否构成一个自洽的 engineering-shear 闭环？请用纯 shear 单元应变和手工 toy tensor 验证。
4. BEC 的 atom mean projection 是否符合训练目标的最佳做法？真实 BEC ASR residual 应如何处理/加权？
5. JARVIS source archive 和 GMTNet `atoms` 的 identity 校验是否足够；对等价原子重排、原胞/常规胞改变应当如何判断？

### B. 对称、PBC 和 strict completion

1. `pymatgen.SpacegroupAnalyzer` 的 cartesian operation 和当前 lattice 行向量 convention 是否匹配；`transformed_cartesian @ inverse_lattice` 是否在所有非正交格下正确？
2. Hungarian element-wise mapping、symprec、`2e-5 Å` mapping tolerance 是否合理；permutation 的 source/target 方向是否与 `transform_internal_tensor` 一致？
3. Reynolds projection + acoustic nullspace 的 rank/intersection 实现是否数学正确；observed block 的 flatten order 与 OUTCAR tensor order 是否一致？
4. ionic closure 使用的 `O_delta` 是否会把“补全正确性”与“人为 regularizer”耦合得过强；是否需要同时报告 stable case 的 exact inverse closure？
5. 对 No.187，应该设计怎样的最小 controlled reproducer 来区分 reference-cell、space-group operation、sign/convention 和 parser 问题？

### C. 图网络的等变性与表达力

1. `CartesianLocalEnvironmentEncoder` 在任意 proper/improper rotation、平移、cell basis change、PBC image relabeling 下，scalar/vector/quadrupole 的变换是否严格符合预期？建议写随机旋转和反射的 property tests。
2. channel-only mixing 而非 CG tensor products 是否会遗漏必要的 angular coupling；是否应切换到 e3nn、MACE、NequIP、Allegro、Equiformer 的成熟 representation？
3. `cutoff=5 Å`、最多 32 邻居、mean/degree aggregation 是否对大 cell、层状材料、长程离子耦合产生系统性偏差？
4. `CrystalGlobalContext` 的 reciprocal shell 对 GL(3,Z) lattice basis change 是否真的等价；有限 shell truncation 如何影响 invariance？
5. rank-3 spectral operator 的 origin invariance、镜像 parity、`uniform_shape / ||uniform||^2` 的数值行为是否合理？

### D. 共同能量、响应算子与分支分解

1. 由 directed edges 组装 `Phi/Lambda/K_eta` 时是否 double count；在一个两原子、一条键的 toy system 中，有限差分能否逐项复现代码公式？
2. `energy_learned_strain` 的 correction 是否依旧来自同一个 scalar strain map；其 signed stiffness 是否使能量不稳定、如何施加物理可接受的约束？
3. `O_delta = Phi(Phi^2+delta^2)^{-1}` 是为训练稳定而定义的 filter；在实际 phonon/DFPT 语境下是否适合用于 ionic piezo、dielectric、elastic？有没有更物理的 soft-mode/imaginary-mode 处理方法？
4. `electronic_piezo = direct + spectral_operator` 与 ionic branch 是否存在 double count；background elastic/dielectric 与 shared clamped term 的分解是否闭合？
5. 介电和弹性 background 强制各向同性是否过度限制；这是否令总响应训练错误地把各向异性塞进不合适的分支？
6. `16.02176634`、`180.9512817`、`160.2176634`、体积因子和 sign 的推导是否正确，请从 SI/atomic units 独立推导。

### E. 训练与评估有效性

1. direct-factor/joint 的 loss weighting、50/50 update budget、冻结策略是否是合理诊断；如何设计 validation-only 的更稳健多 seed hyperparameter search？
2. 是否有预训练、cache、normalization、acquisition ranking 或 split construction 的 target leakage 风险？应如何用 unit/integration test 证明没有？
3. active-only threshold、denominator floor、macro/micro aggregation 是否应报告置信区间、bootstrap 或按晶系分层？
4. total TRS 为负而 branch cancellation ratio 接近 0.9 时，最可能的失败机制如何分解：factor direction、amplitude、electronic branch、response operator、标签 convention，还是数据规模？
5. 哪些改进应先做小型受控 DFPT/phonopy validation，再做大规模训练？请按“高可信低成本”排序。
